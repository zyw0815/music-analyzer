# 工作日志 - 2026-07-01

## 项目：音质分析器 (Music Analyzer)

从零搭建一个专业级 Web 音质分析工具，支持多种音频格式的多维度分析。

### 需求分析与设计
- **功能定义**：内置播放器、质量检测（7 维评分）、频谱分析、波形显示、声道分析、音频信息共 6 个模块。
- **技术选型**：Python FastAPI 后端 + React TypeScript 前端，ECharts 做可视化，librosa 做音频分析。
- **UI 设计**：参考 Audio Expert 工具，左侧功能栏切换模块，暗色专业主题。
- **设计文档**：编写完整 Design Spec，包含系统架构、数据流、API 定义、前端布局、评分算法等。

### 实施计划
- 制定 10 个 Task 的实施计划，每个 Task 遵循 TDD（测试驱动开发）。
- 每个 Task 包含：写测试 → 验证失败 → 实现 → 验证通过 → 提交。

### GitHub 工程化
- **仓库**：https://github.com/zyw0815/music-analyzer
- **分支策略**：main（稳定发布）+ develop（开发主线）+ feature/fix 分支
- **全流程**：每个变更走 Issue → Branch → PR → Review → Merge 完整流程。
- **CI/CD**：GitHub Actions 自动运行后端测试和前端构建。
- **Issue/PR 模板**：标准化 issue 和 PR 的提交格式。

### 后端实现（6 个分析模块）
- **BasicInfoModule**：提取文件属性、音频参数、时长、响度、标签、DSD 信息。使用 mutagen + librosa。
- **QualityModule**：7 维度评分（码率/完整度/质量检测/声道/频谱/动态范围/失真），加权计算综合分，中文等级标签。
- **SpectrumModule**：FFT 频谱分析、STFT 频谱热力图、5 频带频率分布。
- **WaveformModule**：降采样波形、RMS 包络、削波检测（连续 3 个采样点 ≥0.999）、静音检测（RMS < -60dB 持续 > 0.3s）。
- **ChannelModule**：相位相关系数、L/R/Mid/Side 频谱、立体声宽度、单声道检测。
- **PlayerModule**：音频转 WAV 流式播放，支持 DSD 通过 ffmpeg dsd2pcm 转换。
- **API 端点**：POST /api/analyze（全量分析）、POST /api/analyze/basic（快速元数据）、GET /api/stream/{file_id}（播放流）。
- **测试**：38 个单元测试全部通过。

### 前端实现（6 个 UI 模块）
- **侧边栏**：6 个功能导航项，可折叠，active 高亮。
- **顶栏**：文件信息 + 内置播放器（HTML5 audio + 进度条 + 音量）。
- **质量检测**：ECharts 仪表盘 + 7 个子评分进度条 + 可展开评分明细。
- **频谱分析**：频谱线图（对数频率轴）+ 频谱热力图 + 频率分布柱状图。
- **波形显示**：可缩放波形 + RMS 包络 + 削波/静音区域标记。
- **声道分析**：L/R 频谱叠加 + 相位相关仪表 + Mid/Side 频谱。
- **音频信息**：6 卡片信息网格（文件属性、音频参数、时间、响度、标签、DSD）。
- **ErrorBoundary**：防止组件崩溃时白屏，显示友好错误提示。

### Bug 修复
- **端口冲突**：9210 被 QQ 占用，改为 9220。
- **ECharts 黑屏**：频谱热力图 150 万数据点导致浏览器崩溃，降采样到 4 万点。
- **上传 500 错误**：axios 手动设置 Content-Type 缺少 boundary，移除让 axios 自动处理。
- **waveform 缺少 times 字段**：后端返回格式与前端不匹配，添加 times 数组。
- **frequency_distribution 格式不匹配**：后端返回对象数组，前端期望分开的两个数组。
- **图表只显示右上角**：ECharts 组件缺少 width: 100%，添加后图表适配容器。
